- 코드 : 요구사항을 표현하는 언어
- 깨끗한 코드란?
    - 한가지에 집중
    - 중복 피함
    - 좋은 가독성
        
        : 작성자가 아닌 사람도 읽기 쉽고 고치기 쉬움.
        
    - 주의 깊게 작성한 코드
        
        : 누군가 시간을 들여 깔끔하고 단정하게 정리.
        
    - 한 기능만 수행
        
        : 메서드가 여러 기능을 수행한다면 기능을 명확히 기술하는 메서드 하나와 기능을 실제로 수행하는 메서드 여러개로 나누어야 함.
        
    - 작게 추상화
    
- 우리는 코드를 작성하는 저자
    
    저자에게는 독자와 잘 소통할 책임 존재.
    
    새 코드를 짜면서 끊임없이 기존 코드를 읽음.
    
    주변 코드가 읽기 쉬우면 새 코드를 짜기도 쉬움.
    
- 코드는 지속적인 개선이 필요.
